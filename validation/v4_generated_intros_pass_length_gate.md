# 三领域实战验证 V4：通过长度门槛的完整稿

V3 仍未达到 1,500 中文字符的最低门槛，因此本轮按更新后的 skill 重新生成 V4。V4 仍然不作为投稿终稿，因为正式投稿需要用户提供具体综述证据和引用；它用于验证 skill 是否能在几个不同领域生成具有足够内容承载力的 Nature-style omit-results Introduction。

## V4-1：发光/磷光材料文本挖掘

发光材料的高效发现依赖于把分散的实验报道转化为可比较的 host-dopant-emission 关系，而不是只积累更多孤立的发射峰数值。对于磷光材料，真正有用的数据对象是 evidence-linked phosphor sample record：它需要同时绑定主晶格、激活离子及价态、掺杂浓度、激发/发射波长、效率指标和测试条件[基准来源：58Z3MKQN；需补文献]。这一要求来自发光机制本身。主晶格决定局域配位环境和晶场强度，激活离子的价态影响能级跃迁，浓度变化可能引入浓度猝灭或能量迁移，测试温度和激发波长又会改变谱图解释。尤其在多元素主晶格、共掺杂和缺陷调控体系中，单个字段的含义只有放回样品整体配方和测试语境中才成立。因此，数据驱动发光材料设计所需要的并不是“更多光谱数值”，而是能把这些变量放回同一样品语境中的结构化关系。

这类关系已经大量存在于已发表文献中，但论文写法会主动拆散它们。一个样品的化学式和掺杂浓度可能出现在正文的合成叙述或表格中，发射峰、激发波长和半峰宽常位于光谱图、图注或补充信息，量子效率、热猝灭和测试气氛又可能被放在表征段落或应用讨论里。更复杂的是，发光材料论文经常连续讨论一个样品系列：第一句定义主晶格，第二句改变掺杂浓度，第三句比较发射峰位，图中再展示不同激发条件下的谱线。许多文章还会把“最佳样品”的性能放在图注或讨论段中，而把完整系列放在补充表格里；如果抽取流程不能追踪样品编号、系列命名和图谱位置，最关键的性能值反而最容易脱离上下文。对人类读者来说，这些跨句关系可以通过上下文恢复；对自动抽取系统来说，如果没有样品身份和证据位置，每个数值都可能脱离其科学条件。文献因此是潜在数据源，却不是天然数据库。

已有数据获取路线各有价值，但都没有消除关系绑定这一根本瓶颈。人工整理能够保证小规模数据的可信度，却难以覆盖快速增长的发光材料文献，也难以系统记录未报告字段、异常样品和测试条件。领域数据库便于存放已经标准化的发射波长或材料组成，却通常无法恢复论文叙述中的隐含主语、浓度序列和图谱归属。通用化学 parser 与 NER 可以识别元素名、化学式和数值，但它们更擅长实体边界识别，而不是判断一个数值究竟是掺杂浓度、激发波长、发射峰还是处理温度。关系抽取模型可以补充部分字段连接，但通常依赖标注语料，且很难覆盖不同期刊中化学式、省略主语、谱图编号和表格布局的变化。即使引入领域语言模型，如果缺少明确 schema、样品级状态跟踪和原文证据回链，模型仍可能把相邻样品、相邻谱线或相邻表格行合并为一条看似完整的记录。剩下的挑战不是识别“磷光材料”这个实体，而是恢复同一样品内部的光学证据链。

磷光材料尤其放大了误绑定的科学后果。相同的 Eu、Ce、Mn 或 Bi 掺杂符号，在不同价态、不同配位环境和不同浓度下可能对应完全不同的发光中心；同样位于蓝光或红光区的发射峰，只有绑定激发波长、测试温度、主晶格和谱图归属后才具有比较意义。如果抽取系统把某个样品系列的最佳发射强度分配给错误浓度，模型会错误地学习浓度-性能关系；如果把图注中的发射峰绑定到相邻主晶格，数据库会把本来不可迁移的结构效应伪装成普遍规律。更隐蔽的错误来自“看似合理”的补全：当论文没有明确给出价态、测试温度或谱图归属时，模型若按常见发光中心自动填充，就会把假设写成事实。这样的错误不会像缺失字段那样显眼，却会持续污染机器学习训练集，并把后续材料筛选引向虚假的设计方向。因此，发光材料文本挖掘必须把“关系是否被证据支持”放在“字段是否被抽到”之前。

更合适的路径是构建受领域 schema 约束、带证据回链的 phosphor sample record 抽取流程，使 LLM 的上下文理解能力服务于关系恢复，而不是服务于自由摘要。该流程应要求每条记录同时给出主晶格、激活离子及价态、掺杂浓度、激发/发射波长、效率指标、测试条件和原文位置，并能追踪这些字段来自正文、表格、图注还是补充信息。它还应保留样品系列之间的继承关系，使“同上条件”“最佳样品”和“对照样品”不会在抽取时断裂。当价态、浓度、谱图归属或测试条件缺少证据时，模型应保留缺失或不确定标记，而不是依据材料常识补全。由此得到的记录才能同时满足专家复核、跨论文比较和机器学习建模的要求。`需补文献`
这一路径也为后续实验验证保留清楚的数据来源。

## V4-2：有机合成文献挖掘

有机合成的智能化发现首先受限于 reaction record 的获得，而不是受限于模型是否能够生成新的反应建议。一个可用于机器学习、路线规划或知识发现的反应记录，必须同时绑定底物、产物、试剂、催化剂、溶剂、温度、时间、产率、结构表示和表征证据[基准来源：UAL253B7；需补文献]。这些字段并不是独立清单，而是共同定义一个反应事实：同一种化合物在不同语境下可以承担不同角色，同一条件值也可能属于不同步骤。若缺少角色、步骤和证据来源，反应记录就无法支持可验证的反应规律学习。更重要的是，合成知识常常依赖负面空间：哪些底物没有被测试、哪些条件只适用于某一类取代基、哪些步骤属于优化而非通用方法，都会影响模型对反应可迁移性的判断。有机合成文献的价值巨大，正因为其中积累了海量实验事实；它的瓶颈也正在于这些事实并未以机器可读的反应对象出现。

有机合成论文中的反应知识天然是多模态和跨位置的。分子结构和编号常位于反应式图中，箭头上下方记录催化剂、溶剂、温度或时间，实验步骤详细描述加料顺序、后处理和纯化，表格展示底物拓展与产率，补充信息又保存表征数据和失败或变体实验。一个完整 reaction record 因此必须跨越图像、结构、文本和表格：它不仅要知道某个分子被提到，还要判断它在该反应中是底物、产物、催化剂、添加剂还是溶剂；不仅要抽取一个产率，还要知道这个产率对应哪个底物编号和哪组条件。许多反应论文还会在正文中讨论代表性底物，在表格中列出取代基范围，在补充信息中给出具体实验量；如果三者不能被合并，同一个反应事实会被拆成多个不完整片段。对化学家来说，图文对齐往往是阅读反应的基本动作；对自动抽取系统来说，这正是最容易断裂的证据链。

现有数据获取路线都解决了局部问题，却没有形成稳定的跨模态反应记录。商业数据库便于查询规范化反应，但受覆盖范围、字段体系和许可限制影响，难以满足特定反应类型、定制参数或最新文献的需求。高通量实验可以产生格式一致的数据，却成本高、范围窄，无法替代历史文献中已经积累的反应知识。OCSR 能从图像中恢复分子结构，却不保证分子角色、反应编号和实验步骤已经对齐；传统 NLP 或规则系统能抽取文本中的温度、时间和产率，却容易被不同论文风格、编号省略、多步反应和表格布局击穿。监督式关系抽取可以在特定语料上提高字段连接能力，但需要昂贵标注，且很难覆盖反应图式、实验步骤和底物拓展表之间的跨模态关系。LLM 具备上下文解释能力，但如果没有化学结构工具、字段约束和证据定位，也可能把相邻反应、相似编号或多步实验混成一条记录。真正未解决的是 reaction role、condition 和 structure 的共同绑定。

这种绑定错误会直接改变反应事实本身。一个化合物可能在主反应中是底物，在优化实验中是添加剂，在另一个例子中又成为产物；同一个温度可能描述反应阶段、淬灭处理、纯化步骤或表征条件；一个产率可能对应图中编号 3a，也可能对应表格中另一条底物拓展。若抽取系统只识别到化合物名和数值，却不能把结构图中的编号、步骤文本中的条件和表格中的产率合并为同一反应，后续模型将学习到错误的底物适用性和条件-产率关系。在多步合成中，这种错误还会把中间体、目标产物和副产物混淆，使路线规划模型误判可行步骤。比缺失更危险的是错误完整：一条字段齐全但角色错位的反应记录，会在数据库中显得可靠，却会误导下游路线推荐和反应知识发现。

因此，有机合成文献挖掘需要受化学 schema 约束的多模态 LLM 工作流，输出带有结构标识、角色标签、条件字段、产率和证据位置的 reaction record。LLM 应用于解释上下文、处理写作差异和连接跨句信息，OCSR 与化学标准化工具用于约束分子结构和名称归一，RAG 或证据回链用于把每个字段定位回反应式、表格或实验步骤。该工作流还应允许研究者按反应类型定制字段，例如加入选择性、对映体过量、催化剂负载量或底物范围标签。当结构编号、条件归属、角色判断或产率证据不足时，系统应保留不确定性，而不是生成看似完整的反应表；当论文只给出代表性步骤而没有完整底物范围时，系统也应区分 general procedure、substrate scope 和 individual example。这样的反应记录才可能成为可审计、可更新、可定制的合成知识基础，并让研究者区分真实文献证据、标准化转换结果和模型无法确认的空缺字段。`需补文献`

## V4-3：冶金 P-M-S-M-P 系统图抽取

冶金材料文献挖掘的关键对象不是单个工艺名、组织术语或性能数值，而是能够表达 processing-mechanism-microstructure-mechanism-property 的 P-M-S-M-P system chart。结构材料的强度、塑性、低温韧性、抗蠕变能力和相稳定性，往往由加工历史、相变路径、析出行为、晶粒结构、位错机制和测试环境共同决定[基准来源：F9NLVIKW；需补文献]。如果这些环节被拆成孤立字段，材料设计所依赖的因果解释就会丢失。尤其在高熵合金、先进钢和耐高温合金中，同一性能提升可能来自多个机制的叠加或竞争，简单数值表无法表达这种机制结构。更重要的是，冶金论文常把“为什么有效”放在机制讨论而不是数据表中；忽略这一层，抽取结果就只能支持统计汇总，不能支持设计推理。面向合金设计的文本挖掘因此不能只追求“抽到数值”，而要恢复论文中关于加工如何改变组织、组织如何通过机制影响性能的链式解释。

冶金论文恰好以分散形式保存了这种机制链。加工路线和热处理参数常在实验方法中描述，显微组织、析出相和晶粒特征出现在图像、图注和表征段落中，力学性能值位于拉伸、硬度、冲击或低温测试表格，机制解释则分布在讨论部分。许多文章还会把一个机制放在图像解释中，把另一个机制放在性能讨论中，最后在结论里给出综合判断；这些位置对人类读者是连贯叙事，对抽取系统却是跨段落关系。作者在单篇论文中可以自然地把加工条件、组织演化、机制推断和性能变化连成叙事，但自动抽取如果只识别实体边界，就很难判断某一组织特征由哪个加工步骤导致，又通过哪一机制影响哪项性能。系统图的价值在于把这种叙事压缩为可比较的关系对象，同时保留每条关系的证据来源和方向。

现有自动化路线容易在这里失去科学结构。人工阅读可以理解单篇论文的机制叙事，但无法把大量论文持续转化为统一、可绘图、可检索的系统图。常规 NER 和表格抽取能找到“时效处理”“纳米析出物”“屈服强度”等实体和值，却不能可靠确定 processing、mechanism、microstructure 和 property 之间的方向，也无法判断同一机制是否来自作者文本、图像解释还是抽取模型的外部知识。知识图谱方法能够表达关系，但如果关系类型没有嵌入材料机制框架，仍可能把“相关出现”误写成“因果作用”。摘要型 LLM 能生成流畅概述，但往往把文本证据与常识性材料学推断混在一起，缺少可审计的边来源。固定规则虽然可用于格式稳定的论文，却难以跨越实验、计算和混合研究的写作差异。未解决的核心问题不是缺少冶金术语识别，而是缺少保持机制方向、证据来源和关系完整性的系统图抽取。

冶金系统图的错误绑定会改变材料机制解释。相同的强度提升可能来自析出强化、细晶强化、固溶强化、相变诱导塑性或多机制耦合；相同的时效温度在不同合金体系中可能导致完全不同的析出相和相变路径。如果系统把性能提升直接连接到加工条件，而丢失中间组织和机制，读者就无法判断该工艺是否可迁移到另一合金体系；如果模型把外部常识推断出的机制标记为原文证据，系统图看似完整，却会误导后续知识综合。更严重的是，错误机制链会影响材料设计假设：研究者可能把本应优化析出相分布的问题误判为调节晶粒尺寸的问题。对冶金文献而言，机制来源、关系方向和缺失关系与数值本身一样重要。一个好的抽取系统必须知道何时连接节点，也必须知道何时不连接。

因此，更合适的文本挖掘路径是以 P-M-S-M-P schema 约束 LLM，把输出限制为带证据标签、缺失关系标记和机制来源说明的 auditable system chart。模型需要从论文中识别加工、机制、组织、机制和性能节点，并为每条边标注其来自正文、图注、表格、作者讨论还是外部知识；当论文没有给出明确机制时，系统应保留空缺或标注推测来源，而不是自动补成完整因果链。这样的系统图还应保留边的方向和层级，使“加工导致组织变化”和“组织解释性能提升”不会被压平为普通相关关系。它还应允许同一性能对应多条机制路径，因为冶金材料中的强化、塑化和相稳定性常由多个机制共同贡献。由此得到的系统图既能服务单篇论文理解，也能作为跨论文材料发现和知识综合的中间表示，并让专家在后续审核中快速定位最可能影响材料设计判断的薄弱关系。`需补文献`
这类表示还可以把不同论文中的机制链放到同一比较框架中，帮助识别可迁移设计原则和仍需人工复核的薄弱证据，并支撑后续实验优先级判断。

## V4 验证结论

V4 才真正满足本轮实战验证要求：三个领域都形成了完整五段 Introduction，且都包含 early output object、prior-route pressure chain、material/task-specific mis-binding consequence、domain-specific method object。相比 V1 的五段式模板和 V2/V3 的偏短问题，V4 更接近高质量基准文章的承载方式。
